### Команды в GIT Bash для Windows:
1. ```pwd``` (print working directory) - показать текущую директорию/папку
2. ```cd``` (change directory) - изменить текущую директорию. После cd указывается куда перейти<br>
Примеры:<br>
2.1 ```cd ~``` переход в домашнюю директорию<br>
2.2 ```cd /d``` - переход на диск d<br>
2.3 Если в названии есть пробелы то нужно поставить кавычки (любые) ```cd "Saved Games"```<br>
2.4 ```cd ..``` - переход на директорию выше<br>
2.5 Можно перемещаться сразу через несколько директорий: ```cd projects/github/open-source<br>```
3. ```ls``` (list directory contents) - показать содержимое директории<br>
Примеры:<br>
3.1 Просто ```ls``` - выведет инфу текущей директории<br>
3.2 ```ls -a``` (```-a``` это флаг/ключ, доп функционал команды) выведет все файлы, в том числе и скрытые<br>
3.3 ```ls``` аналогично работает с ```~, .. , pwd``` или можно вывести инфу определенной (не текущей) директории ```ls projects/github/open-source```<br>
4. ```touch``` - создание файла. Указывается сразу с названием файла. Хорошая практика указывать сразу расширение файла. Пример - ```touch test.txt```<br>
4.1 Можно создать файл сразу в нужной директории: ```touch dir1/dir-inside/dir-deeper-inside/test.txt```<br>
4.2 Можно создать файл на две директории выше: ```touch ../../file.txt```<br>
5. ```mkdir``` (make directory) - создание директории/папки. Указывается сразу с названием директории/папки: ```mkdir new-project```<br>
5.1 Можно сразу создать структуру из нескольких директорий через флаг ```-p```: ```mkdir -p dir1/dir-inside/dir-deeper-inside```<br>
В целом и ```touch``` и ```mkdir``` работают и с ```~``` и ```..```
6. ```cp``` - копирование файлов. Указывается как cp + что копируем + куда копируем. Можно указывать сразу несколько файлов:<br>
6.1 ```cp index.html style.css script.js src/``` (скопировали три файла (index.html, style.css и script.js) в папку src)
7. ```mv``` (move) - перемещение файлов. Синтаксис аналогичен cp<br>
7.1 ```mv table.csv ./very-important-files``` (сначала указываем имя файла, который хотим переместить, потом путь куда перемещаем)
8. ```cat``` (concatenate and print) - вывод содержимого txt файла. ```cat test.txt```
9. ```rm``` (remove) - удаление файла: ```rm test.txt```
10. ```rmdir``` (remove directory) - удаление директории: ```rmdir project```. Удаляет только если директория пустая!
11. ```rm -r``` (-r recursive) - удаление непустой директории. Удаляет рекурсивно, т.е. последовательно каждый элемент папки, затем саму папку
12. ```&&``` - двойной амперсанд объединяет несколько команд в один запрос:<br>
12.1 ```mkdir second-project && cd second-project && touch index.html style.css``` (создаём папку second-project, переходим в папку second-project и создаём в ней два файла: index.html и style.css)

### Команды для навигации/ускорения:
1. ↑↓ - все введенные команды хранятся в буфере. Их можно быстро выводить через стрелочки
2. Tab - можно "дописывать" путь/название файла когда не знаешь точного или лень

### GIT
1. На Windows ставится сразу с GIT For Windows вместе с командной строкой
2. Сетап инфы о себе для команды<br>
	```git config --global user.name "User Namovich"```<br>
	```git config --global user.email username@yandex.ru```<br>
3. ```cat ~/.gitconfig``` - настройки гита
4. ```git init``` - установка папки как git-репы.<br>
```rm -rf .git``` - "разгитить" папку. -r рекурсирвное удаление, f скип подтверждений удаления)
5. ```git status``` - проверка репозитория. Показывает название ветки (master/main), инфу по коммитам, инфу по текущим версиям файлов<br>
Зеленые файлы - последние что были фиксированы через git add. красные - были с изменениями, но которые еще не фиксированы. 
6. ```git log``` - история коммитов: кто, когда, с каким комментарием добавлял.

### SSH
Сетевой протокол для безопасного обмена данными в сети. Создает пару ключей (приватный и публичный). Приватный ключ расшифровывает публичный, должен храниться только локально на компе и его **НЕЛЬЗЯ** никому передавать! Публичный доступен всем
1. ```cd ~ && ls -la .ssh/``` проверка наличия ssh ключей на компе (примеры: id_dsa.pub, id_ecdsa.pub, id_rsa, id_ed25519.pub)
2. ```ssh-keygen -t ed25519 -C "почта с гитхаба"``` создание пары SSH-ключей. ed25519 это алгоритм шифра, если не подходит то можно взять иной: ```ssh-keygen -t rsa -b 4096 -C "почта с гитхаба"```
При создании пары ключей спросит добавить или нет passphrase. Добавлять по своему желанию
3. Добавление **ПУБЛИЧНОГО** ssh ключа на github: скопировать содержимое .pub ключа, в настройках на github в SSH and GPC keys добавить новый SSH key
4. ```ssh -T git@github.com``` - проверка добавленного ssh ключа. При первом разе нужно проверить ответ от GitHub'а на [этом сайте](https://docs.github.com/en/authentication/keeping-your-account-and-data-secure/githubs-ssh-key-fingerprints) чтобы убедиться что сервер корректный 


### Настройка Github
Это платформа для хранения it-проектов и командной работы над ними. На ней можно фиксировать свой локальный репозиторий на удаленном.
1. Регаемся на https://github.com
2. Создаем удаленный репозиторий через свой профиль. Имя удаленной репы желательно (но не обязательно) называть также как локальную. На странице репы в разделе "Quick setup" выбираем ssh и копируем значение.
3. Связывание удаленной и локальной репы<br>
3.1 Переходим через cd в локальную репу<br>
3.2 ```git remote add origin "копипаста ssh значения из Quick setup"```<br>
3.3 ```git remote -v``` проверка что репы связались. ```-v``` сокращение от ```--verbose``` (подробный)


## Цикл коммита:
1. ```git init``` - делаем папку локальной репой. ```rm -rf .git``` - "разгитить" папку если нужно.
2. Добавляем/изменяем нужные файлы в локальной репе
3. ```git add file_name или git add -all``` - подготовка нужных файлов для отслеживания версий
4. ```git commit -m "комментарий по изменениям"``` - коммитимм файлы, ```-m``` ключ для добавления комментария<br>
```git add``` - фиксирует текущее состояние файлов. Аналог добавления товаров в корзину в онлайн магазине<br>
```git commit``` - сохранение текущих файлов. Покупка товаров из корзины
5. ```git push origin main/master``` - связываем локальную и удаленную репы. В первый раз нужно будет добавить -u: ```git push -u origin main/master```
origin - имя удаленный репы. main/master - название ветки
6. Историю коммитов можно глянуть в удаленной репе
