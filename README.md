## Цикл коммита:
0. ```git init``` - инициализировать репу. ```rm -rf .git``` - "разгитить" папку если нужно.<br>
0.1 ```git remote add origin <git_hub_link>``` - привязать локальную репу к удаленной *(не забудь про ssh!)*.<br>
0.2 ```git remote -v``` - проверить что репы связались<br>
1. Добавляем/изменяем нужные файлы в локальной репе (старые файлы станут ```modified```)<br>
2. Подготовка нужных файлов для отслеживания версий<br>
Аналог добавления товаров в корзину в онлайн магазине<br>
```git add file_name``` или ```git add -all``` (все ```modified``` и новые файлы) или ```git add .```(всю текущую папку)<br>
3.```git commit -m "комментарий"``` - коммитимм файлы, ```-m``` ключ для добавления комментария<br>
Аналог покупки товаров из корзины<br>
4.```git status``` - проверить состояние файлов в репе<br>
5. ```git push origin master/main``` - связываем локальную и удаленную репы. В первый раз нужно будет добавить -u: ```git push -u origin main/master```<br>
origin - имя удаленный репы. master/main - название ветки<br>

### Изменение коммитов
0.1 ```git --log``` или ```git --log oneline``` история коммитов<br>
0.2 ```git diff <хеш A> <хеш B>``` - сравнить разницу двух разных коммитов<br>
0.3 ```git diff --staged``` - сравнить разницу в ```staged``` файлах<br>
1. ```git commit --amend``` - изменение *поледнего* (HEAD) коммита. Например забыл добавить файл, сделал ```git add``` а потом эту команду<br>
1.1 ```git commit --amend -m "text"``` - изменение коммента *последнего* коммита.<br>
Можно писать также ```--amend --no-edit``` чтобы оставить коммент без изменения<br>
2.```git restore <file>``` - откатить ```modified``` файл до последней сохраненной версии<br>
3.```git restore --staged <file>``` - убрать файл из отслеживания (т.е. из ```git add``` сделать ```unstage```<br>
4.```git reset --hard <хеш_коммита>``` - откатить все изменения до указанного коммита.<br>
*Указывать коммит к которому сделать откат, а не который удалять!*<br>


### Статусы файлов в GIT
* ```untracked``` - файл в репе, но не отслеживается
* ```staged``` - файл подготовлен к коммиту через ```git add```<br>
* ```modified``` - файл был изменен по сравнению с последней версией<br>
Впервые добавленный файл будет помечен как ```new file: name```<br>
*Один и тот же файл может быть одновременно и в staged (готов к коммиту) и в modifed (изменен). Значит это две разные версии одного файла!*<br>
Иначе говоря: ```modified``` + ```git add``` = ```staged```

### Команды в GIT Bash для Windows:
1. ```pwd``` (print working directory) - показать текущую директорию/папку
2. ```cd``` (change directory) - изменить текущую директорию. После cd указывается куда перейти<br>
Примеры:<br>
2.1 ```cd ~``` переход в домашнюю директорию<br>
2.2 ```cd /d``` - переход на диск d<br>
2.3 Если в названии есть пробелы то нужно поставить кавычки (любые) ```cd "Saved Games"```<br>
2.4 ```cd ..``` - переход на директорию выше<br>
2.5 Можно перемещаться сразу через несколько директорий: ```cd projects/github/open-source<br>```
3. ```ls``` (list directory contents) - показать содержимое директории<br>
Примеры:<br>
3.1 Просто ```ls``` - выведет инфу текущей директории<br>
3.2 ```ls -a``` (```-a``` это флаг/ключ, доп функционал команды) выведет все файлы, в том числе и скрытые<br>
3.3 ```ls``` аналогично работает с ```~, .. , pwd``` или можно вывести инфу определенной (не текущей) директории ```ls projects/github/open-source```<br>
4. ```touch``` - создание файла. Указывается сразу с названием файла. Хорошая практика указывать сразу расширение файла. Пример - ```touch test.txt```<br>
4.1 Можно создать файл сразу в нужной директории: ```touch dir1/dir-inside/dir-deeper-inside/test.txt```<br>
4.2 Можно создать файл на две директории выше: ```touch ../../file.txt```<br>
5. ```mkdir``` (make directory) - создание директории/папки. Указывается сразу с названием директории/папки: ```mkdir new-project```<br>
5.1 Можно сразу создать структуру из нескольких директорий через флаг ```-p```: ```mkdir -p dir1/dir-inside/dir-deeper-inside```<br>
В целом и ```touch``` и ```mkdir``` работают и с ```~``` и ```..```
6. ```cp``` - копирование файлов. Указывается как cp + что копируем + куда копируем. Можно указывать сразу несколько файлов:<br>
6.1 ```cp index.html style.css script.js src/``` (скопировали три файла (index.html, style.css и script.js) в папку src)
7. ```mv``` (move) - перемещение файлов. Синтаксис аналогичен cp<br>
7.1 ```mv table.csv ./very-important-files``` (сначала указываем имя файла, который хотим переместить, потом путь куда перемещаем)
8. ```cat``` (concatenate and print) - вывод содержимого txt файла. ```cat test.txt```
9. ```rm``` (remove) - удаление файла: ```rm test.txt```
10. ```rmdir``` (remove directory) - удаление директории: ```rmdir project```. Удаляет только если директория пустая!
11. ```rm -r``` (-r recursive) - удаление непустой директории. Удаляет рекурсивно, т.е. последовательно каждый элемент папки, затем саму папку
12. ```&&``` - двойной амперсанд объединяет несколько команд в один запрос:<br>
12.1 ```mkdir second-project && cd second-project && touch index.html style.css``` (создаём папку second-project, переходим в папку second-project и создаём в ней два файла: index.html и style.css)

### Команды для навигации/ускорения:

### SSH
Сетевой протокол для безопасного обмена данными в сети. Создает пару ключей (приватный и публичный). Приватный ключ расшифровывает публичный, должен храниться только локально на компе и его **НЕЛЬЗЯ** никому передавать! Публичный доступен всем
1. ```cd ~ && ls -la .ssh/``` проверка наличия ssh ключей на компе (примеры: id_dsa.pub, id_ecdsa.pub, id_rsa, id_ed25519.pub)
2. ```ssh-keygen -t ed25519 -C "почта с гитхаба"``` создание пары SSH-ключей. ed25519 это алгоритм шифра, если не подходит то можно взять иной: ```ssh-keygen -t rsa -b 4096 -C "почта с гитхаба"```
При создании пары ключей спросит добавить или нет passphrase. Добавлять по своему желанию
3. Добавление **ПУБЛИЧНОГО** ssh ключа на github: скопировать содержимое .pub ключа, в настройках на github в SSH and GPC keys добавить новый SSH key
4. ```ssh -T git@github.com``` - проверка добавленного ssh ключа. При первом разе нужно проверить ответ от GitHub'а на [этом сайте](https://docs.github.com/en/authentication/keeping-your-account-and-data-secure/githubs-ssh-key-fingerprints) чтобы убедиться что сервер корректный

### Общая инфа в ```git log```
* Хеширование<br>
Это способ преобразовать набор данных и получить их "отпечаток" (fingerprint).<br>
GIT хеширует коммиты через SHA-1 (Secure Hash Algorithm). Даже если различие будет всего в один символ, хеш будет уже совершенно другой.
* Инфа о коммите это:<br>
когда и кем был сделан коммит, его содержание + ссылка на дочерний или родительский коммит.
* ```git log -oneline``` сокращенная версия инфы о коммите:<br>
Содержит максимум *72* символа!<br>
Короткий, но уникальный хеш в пределах репы который можно использовать как основной. Не обязательно 7 симмволов!<br>
Комментарий, если был указан<br>
В команде важно договориться о том как оформляете комментарии! Например через тикеты Jira
* ```HEAD```<br>
Это служебный *файл* который содержит ссылку на содержание последнего коммита

### Игнорирование файлов в .gitigonre
Это текстовый файл в репе в котором по одному имени на строку указывают файлы которые нужно игнорировать<br>
* Применяется только к новым (untracked) файлам!*<br>
* ```git status --ignored``` Отображение статуса игнорируемых файлов. ```cat .gitignore``` отображение что игнорируется*<br>
- ```# текст``` - указать комментарий<br>
- ```.DS_Store``` - игнорировать определенный файл<br>
- ```*.jpeg``` - звездочка означает любую строку. Т.е. все файлы формата .jpeg будут проигнорированы<br>
- ```**``` - двойная звездочка = игнор всех подпапок между корневой и конкретным файлом<br>
- ```docs/*/tmp``` -  игнорировать все файлы "tmp" во всех подпапках папки docs<br>
- ```file?.txt``` - ? = 1 любой смимвол. filaA.txt будет проигнорирован, fileAA.txt уже нет<br>
- ```[...]``` - игнор файлов с определенными символами. ```[0-9]```,```[abc]```,```[a-z]```<br>
Например ```file[0-2].txt``` проигнорит file0.txt, file1.txt, file2.txt<br>
- ```/``` - слеш указывает на каталоги/папки. Например ```/todo.txt``` пропустит todo.txt в корне репы. ```build/``` применится ко всей папке build<br>
- ```!``` - ! используются для исключений. Например есть ```*.jpeg``` и ```!mem.jpeg```. Все файлы .jpeg игнорятся кроме mem.jpeg
